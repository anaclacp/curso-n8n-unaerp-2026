# Para se aprofundar

Este roteiro reúne materiais para continuar estudando depois do minicurso. A ideia não é ler tudo de uma vez, mas ter um caminho: começar pelos fundamentos, praticar com n8n e APIs, e depois entrar em agentes, contexto, documentação técnica e papers.

> Uma dica para se destacarem no meio de tantos profissionais: esse material de estudos é mais do que decorar ferramentas, a intenção é desenvolver critério de engenharia: saber o que usar, quando usar e por que usar. Essa é uma diferença importante entre apenas receber ordens e executar tarefas, e atuar como uma pessoa engenheira que pensa no problema, avalia alternativas, entende custos e toma decisões técnicas com responsabilidade.

## Como usar este roteiro

1. Comece pelo [básico](./basico/README.md) se você ainda está ganhando segurança com APIs, HTTP, JSON e automações.
2. Vá para o [intermediário](./intermediario/README.md) quando já conseguir montar fluxos, chamar serviços externos e ler documentação.
3. Use o [avançado](./avancado/README.md) para estudar papers e ideias mais profundas sobre LLMs, Transformers, agentes e engenharia de contexto.

## Dicas gerais

- Desenvolva critério, não apenas repertório. Conhecer várias ferramentas ajuda, mas o mais importante é entender em que situação cada uma faz sentido.
- Leia documentação oficial. Ela costuma ser mais precisa do que posts soltos, exemplos antigos ou respostas prontas. Quando estiver estudando uma ferramenta, crie o hábito de abrir a documentação antes de procurar atalhos.
- Pratique com projetos pequenos. Um fluxo simples que consulta uma API, salva dados e chama um modelo de linguagem ensina mais do que assistir muitas horas sem implementar.
- Anote conceitos em linguagem própria. Se você consegue explicar o que é uma API, um token, uma tool, um embedding ou um banco vetorial sem copiar a definição, o aprendizado ficou mais sólido.
- Compare ferramentas. n8n, APIs, LLMs, bancos vetoriais e frameworks de agentes mudam rápido. O mais importante é entender os princípios para trocar de ferramenta quando necessário.
